## 3. Main Discussion

### 3.1 Project Topic

The team discussed several possible project ideas and agreed to focus on a portal system. After discussion, the team selected **MyUS** as the project name.

**MyUS** is an AI-powered portal that optimizes course registration and academic management for students, faculty, and administrators. The platform features smart scheduling, personalized wishlists, demand forecasting to prevent overcrowding, and peer study group matching. It bridges administrative efficiency with an enhanced, data-driven digital learning environment.

### 3.2 Project Direction

The team decided that MyUS should be developed as a web application. The portal will support multiple types of users, including students, faculty members, and administrators.

The system is expected to improve the course registration experience by helping students plan their schedules more effectively, while also helping administrators manage course demand and academic resources.

### 3.3 Advanced Features

The team proposed several advanced features for MyUS:

| Feature | Description |
| --- | --- |
| Smart Scheduling | Helps students build suitable timetables and avoid schedule conflicts. |
| Personalized Wishlist | Allows students to save preferred courses before official registration. |
| Demand Forecasting | Predicts course demand to help administrators prevent overcrowded classes. 
| Academic Management Dashboard | Helps faculty and administrators monitor enrollment, course demand, and student activity. |

### 3.4 AI Feature Discussion
The proposed AI features include smart scheduling recommendations and demand forecasting

These AI features are useful because they directly enhance the registration process, reduce timetable conflicts, help administrators plan course capacity, and support student collaboration.
