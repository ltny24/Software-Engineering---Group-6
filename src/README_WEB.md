# MyUS Portal — Project Setup & Running Instructions 

This guide provides step-by-step instructions for setting up the database, configuring the Spring Boot backend, and launching the React frontend on your local machine.

---

# 1. System Requirements

Before starting, make sure the following software is installed:

- Java Development Kit (JDK) **17** or **21**
  ```bash
  java -version
  ```

- Node.js **v18.x** or later
  ```bash
  node -v
  ```

- Maven (Optional – the project already includes Maven Wrapper)

- SQL Server or MySQL

- A database management tool such as:
  - SQL Server Management Studio (SSMS)
  - DBeaver
  - MySQL Workbench

- Git

---

# 2. Clone the Repository

Clone the project to your local machine.

```bash
git clone <repository-url>
cd MyUS
```

---

# 3. Database Setup

## Step 1. Create Database

Open your preferred database management tool and connect to your local database server.

Create a new database if necessary.

Example:

```sql
CREATE DATABASE myus_db;
```

---

## Step 2. Execute the SQL Script

Locate the SQL file inside the project.


```
src/backend/src/main/resources/db/schema.sql
```

and

```
src/backend/src/main/resources/db/mock_data_myus.sql
```

Execute the script completely.

The script will:

- Create all required tables
- Insert sample data
- Insert test accounts

Example tables include:

- Student
- Course
- CourseOffering
- CourseRegistration
- Appeal
- Department
- ...

---

## Important

Passwords stored in the database are **BCrypt hashed**.

**Do NOT manually insert plaintext passwords into the database.**

Spring Security will reject them.

Use the sample accounts included in the SQL script.

---

# 4. Backend Setup (Spring Boot)

## Step 1. Configure Database Connection

Navigate to

```
backend/src/main/resources/
```

Open

```
application.properties
```

(or `application.yml`)

Update the datasource information.

Example:

```properties
spring.datasource.url=jdbc:sqlserver://localhost:1433;databaseName=myus_db;encrypt=true;trustServerCertificate=true

spring.datasource.username=sa

spring.datasource.password=YourPassword
```

Save the file.

---

## Step 2. Run the Backend

Open a terminal.

Navigate into the backend folder.

### Windows

```bash
mvnw.cmd clean install -DskipTests
mvnw.cmd spring-boot:run -DskipTests
```

### macOS / Linux

```bash
mvn clean install -DskipTests
mvn -DskipTests spring-boot:run
```

---

### Successful Startup

The backend is running successfully when the console displays messages similar to

```
Started Application in X seconds

Tomcat initialized with port(s): 8080
```

Backend URL:

```
http://localhost:8080
```

---

# 5. Frontend Setup (React)

Open a **new terminal**.

Do **NOT** close the backend terminal.

Navigate to the frontend folder.

```bash
cd frontend
```

Install dependencies.

Using npm:

```bash
npm install
```

or Yarn:

```bash
yarn install
```

---

Start the development server.

If using Create React App:

```bash
npm start
```

If using Vite:

```bash
npm run dev
```

or

```bash
yarn dev
```

---

Frontend URLs

Create React App

```
http://localhost:3000
```

Vite

```
http://localhost:5173
```

# 7. Troubleshooting

## Problem 1

### Port 8080 already in use

Error:

```
Web server failed to start.

Port 8080 was already in use.
```

### Solution 1

Change the backend port inside

```
application.properties
```

```properties
server.port=8081
```

### Solution 2

Find the process using port 8080.

Windows:

```bash
netstat -ano | findstr :8080
```

Terminate the corresponding Java process using Task Manager.

---

## Problem 2

### Login page is skipped and Dashboard opens immediately

### Cause

An old JWT token is stored inside Local Storage.

### Solution

Open Developer Tools

```
F12
```

Navigate to

```
Application
```

↓

```
Local Storage
```

↓

```
http://localhost:3000
```

Click

```
Clear All
```

Refresh the page.

---

## Problem 3

### DataSource Error

The backend terminates immediately with a datasource error.

### Cause

Incorrect database configuration.

### Solution

Verify:

- SQL Server/MySQL service is running
- Database name is correct
- Username is correct
- Password is correct

---

## Problem 4

### Maven command not found

Install Maven or use the included Maven Wrapper.

Windows

```bash
mvnw.cmd
```

Linux/macOS

```bash
./mvnw
```

---

## Problem 5

### Node modules missing

Delete

```
node_modules
```

and

```
package-lock.json
```

then reinstall.

```bash
npm install
```

---

# 9. Running Order

Always start the project in the following order:

1. Start the database service
2. Execute the SQL script
3. Configure `application.properties`
4. Run the backend
5. Run the frontend
6. Login using the provided test accounts
