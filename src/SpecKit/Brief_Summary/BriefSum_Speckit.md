# A brief summary of Spec Kit training.

**Performance by:** Hồ Thị Như Ngọc

### 1. Execution steps


**Step 1:** Install uv via brew (Mac) or winget (Windows). Then use the CLI command uvx to run the specify tool. This command should be run in an existing project.

![Brief summary evidence](evidence/Ngoc_evidence_SK_1.jpg)

**Step 2:** Type the commands to set up. Including:

1. /constitution - Establish project principles
2. /specify - Create baseline specification
3. /plan - Create implementation plan
4. /tasks - Generate actionable tasks
5. /implement - Execute implementation

![Brief summary evidence](evidence/Ngoc_evidence_SK_2.jpg)

**Step 3:** Review and edit

### 2. Generated files and directories

First are the files and directories from the installation step, including: directories and files from GitHub matching the chosen AI type, the .github/prompt directory containing pre-prepared prompts, and the .specify directory. The .specify directory includes 3 subdirectories as follows: memory/ containing the constitution file, scripts/ containing code snippets for automation, and templates/ containing pre-formatted template files as a basis for filling in information.

![Brief summary evidence](evidence/Ngoc_evidence_SK_3.jpg)

Next are the documents and source code created during the development process, including 1 new Git branch for the feature being developed, 1 specification file providing an overview of the feature, 1 technical plan detailing the tools and construction methods, 1 list of coding tasks, and the source code of the feature.
